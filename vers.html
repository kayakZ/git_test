Изменение кода на GitHub с 07.02.2025

Главное правило использования Git - никогда не производи отладку в Мастер-ветви. Мастер - ветвь содержит только отлаженный код на основе которого генерится конечный программный продукт. Мастер-ветвь аккумулирует в себе все отлаженные дополнения других программистов! Если требуется интеграция вашего кода с кодом других программистов, то на стороне репозитория необходимо создать ветвь на основе мастера, название, к примеру, developer. Каждый разработчик должен работать только с developer. Т.е. каждое дополнение в программу от разных разработчиков заливается в developer. Далее эта ветвь отлаживается и в случае должного функционирования заливается в мастер-ветвь.
